# Issue 正文（复制到描述框）

## 摘要

测速报告里的「延迟」「响应时间」常被误读为「网站快慢」的全部答案。实际上，它代表的是**从检测节点到目标之间，特定协议下的往返或首包时间**。

PING 延迟反映 ICMP 路径；HTTPS 延迟包含 DNS（若适用）、TCP 连接、TLS 握手、服务器处理与首字节返回，更接近页面打开的第一步，但不等于整页加载时间。

**如何解读 SpeedCE 平均延迟：**
- 静态博客：100–300ms 国内常见，可接受；
- API 接口：建议关注 P95 异常节点，而非仅看均值；
- 跨国访问：200–400ms 未必异常，需结合业务容忍度。

更要紧的是地图上的**异常节点比例**：10% 区域超时比「平均 200ms 但全国 sporadic 红点」更好修。用 SpeedCE 同时看**通畅率 + 延迟 + 地图分布**：https://speedce.com/?lang=zh-CN。
