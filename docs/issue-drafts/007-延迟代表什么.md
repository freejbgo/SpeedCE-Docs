# Issue 标题

问：测速延迟 200ms，算快还是慢？

---

# Issue 正文

## 问题

不同工具显示的毫秒数差很多，如何判断对用户有没有影响？

## 回答

延迟是**往返时间（RTT）**，受地理距离、运营商路由、是否走 CDN 影响。

粗参考（国内 HTTPS，非绝对标准）：

- &lt; 80ms：多数场景体验良好
- 80–200ms：可用，偏远省份常见
- &gt; 300ms 或超时：需结合是否「能打开」

**比平均延迟更重要的是：** 有没有省份持续超时？5% 区域全红，对当地用户就是 100% 不可用。
