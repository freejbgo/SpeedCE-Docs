# Issue 标题

问：主站正常但 api 子域打不开，要单独测吗？

---

# Issue 正文

## 问题

`www.example.com` 正常，`api.example.com` 报错。能否用主站结果代替 API？

## 回答

**不能。** 子域有独立的 DNS、证书、Nginx `server_name`、后端进程。

排查清单：

1. 单独 HTTPS 测 `api.` 子域
2. 对比主域地图：主域绿、API 红 → 查子域配置
3. 检查证书 SAN 是否包含该子域
4. 检查 API 是否走了不同 upstream 或端口

维护一份子域清单，变更后逐个验收。
